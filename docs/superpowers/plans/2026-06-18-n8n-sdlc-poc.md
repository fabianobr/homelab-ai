# n8n SDLC Orchestrator — PoC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Provar que n8n Chat + OpenCode + Ollama formam um ambiente de desenvolvimento agêntico local funcional, cobrindo fases cognitivas (Discovery → Spec) e fases de código (Spec → Code) de ponta a ponta.

**Architecture:** Ambiente híbrido com duas superfícies de trabalho:
- **n8n Chat** (browser) para fases cognitivas — conversa multi-turno com o Ollama, salva spec em arquivo compartilhado
- **OpenCode** (terminal TUI) para fase de código — lê a spec, gera código via Ollama, commita no git
- **Hermes/Telegram** para notificações entre fases

**Tech Stack:** n8n 2.23.3 com Chat Trigger, OpenCode (TUI), Ollama via `http://ollama:11434`, qwen3-coder:30b (reasoning/specs), qwen2.5-coder:32b (code gen)

## Global Constraints

- URL do Ollama a partir do n8n: `http://ollama:11434` (container-to-container, NÃO usar localhost)
- URL do Ollama a partir do host/OpenCode: `http://localhost:11434/v1` (OpenAI-compatible)
- API do n8n: `http://localhost:5678/api/v1/` (sem autenticação na versão atual)
- Chat UI do n8n: `http://localhost:5678/webhook/{workflow-chat-path}` (quando workflow ativo)
- Timeout de chamadas ao Ollama: 180s (modelos 30B demoram ~60-90s por resposta)
- Workspace do PoC: `agents/sdlc-poc/` no repositório
- Handoff spec: arquivo salvo em `/tmp/sdlc-poc/spec.md` (acessível por n8n container e host)

### Task 2: Criar e importar Workflow 1 — Chat de Discovery → Spec

**Files:**
- Create: `agents/sdlc-poc/workflows/01-chat-discovery-to-spec.json`
- Create: `agents/sdlc-poc/tests/import-workflows.sh`

**Interfaces:**
- Consumes: mensagens do usuário via n8n Chat UI (multi-turno)
- Produces: interface de chat em `http://localhost:5678/webhook/{id}` + salva `spec.md` em `/tmp/sdlc-poc/spec.md`

**Como funciona na prática:**
1. Usuário abre `http://localhost:5678/webhook/{chat-id}` no browser
2. Digita o problema em linguagem natural (ex: "quero um app de gestão de tarefas pessoais")
3. n8n chama Ollama → retorna discovery estruturado no chat
4. Usuário refina ("adiciona métrica de engajamento semanal")
5. Usuário digita "gerar spec" → n8n gera PRD e salva em `/tmp/sdlc-poc/spec.md`
6. n8n responde no chat com "Spec salva. Abra o OpenCode para implementar."

- [ ] **Step 1: Criar o JSON do workflow de Chat**

Criar `agents/sdlc-poc/workflows/01-chat-discovery-to-spec.json`:

```json
{
  "name": "SDLC PoC 01 — Chat Discovery to Spec",
  "nodes": [
    {
      "parameters": {
        "options": {}
      },
      "id": "1a2b3c4d-0001-0000-0000-000000000001",
      "name": "Chat Trigger",
      "type": "@n8n/n8n-nodes-langchain.chatTrigger",
      "typeVersion": 1.1,
      "position": [240, 300],
      "webhookId": "sdlc-poc-chat"
    },
    {
      "parameters": {
        "jsCode": "const SYSTEM_PROMPT = `You are an AI Product Manager helping run a structured software discovery session.\n\nYou guide the user through these stages in order:\n1. DISCOVERY: understand the problem, context, users affected\n2. HYPOTHESES: formulate 3 testable hypotheses\n3. METRICS: define 3 measurable success KPIs\n4. SPEC READY: when user says 'gerar spec' or 'generate spec', produce the full PRD\n\nWhen producing the PRD (stage 4), use this exact format:\n\n---SPEC-START---\n## Feature Name\n[verb-noun, e.g. \"Manage Personal Tasks\"]\n\n## Background\n[1 paragraph]\n\n## Functional Requirements\nRF-01: The system MUST ...\nRF-02: The system MUST ...\n[5-8 requirements]\n\n## Non-Functional Requirements\n[3-5: performance, security, observability]\n\n## Acceptance Criteria\n[Given/When/Then for each RF]\n\n## Out of Scope\n[3+ items]\n\n## Definition of Done\n[checklist]\n---SPEC-END---\n\nFor all other messages, respond conversationally to refine the discovery. Keep responses concise.`;\n\nconst messages = $input.first().json.messages || [];\nconst userMessage = $input.first().json.chatInput || '';\n\n// Build message history for multi-turn\nconst history = messages.map(m => ({ role: m.role, content: m.content }));\nhistory.push({ role: 'user', content: userMessage });\n\nconst allMessages = [{ role: 'system', content: SYSTEM_PROMPT }, ...history];\n\nconst resp = await fetch('http://ollama:11434/api/chat', {\n  method: 'POST',\n  headers: { 'Content-Type': 'application/json' },\n  body: JSON.stringify({\n    model: 'qwen3-coder:30b',\n    stream: false,\n    options: { temperature: 0.3 },\n    messages: allMessages\n  })\n});\n\nif (!resp.ok) throw new Error(`Ollama error: ${resp.status} ${await resp.text()}`);\nconst data = await resp.json();\nconst reply = data.message.content;\n\n// If response contains a spec, save it to the handoff file\nif (reply.includes('---SPEC-START---')) {\n  const specMatch = reply.match(/---SPEC-START---([\\s\\S]*?)---SPEC-END---/);\n  if (specMatch) {\n    const spec = specMatch[1].trim();\n    // Write spec to handoff file via n8n filesystem\n    const fs = require('fs');\n    fs.mkdirSync('/tmp/sdlc-poc', { recursive: true });\n    fs.writeFileSync('/tmp/sdlc-poc/spec.md', spec, 'utf8');\n  }\n}\n\nreturn [{ json: { output: reply } }];"
      },
      "id": "1a2b3c4d-0001-0000-0000-000000000002",
      "name": "SDLC Chat LLM",
      "type": "n8n-nodes-base.code",
      "typeVersion": 2,
      "position": [480, 300]
    }
  ],
  "connections": {
    "Chat Trigger": {
      "main": [[{ "node": "SDLC Chat LLM", "type": "main", "index": 0 }]]
    }
  },
  "active": false,
  "settings": { "executionOrder": "v1" }
}
```

- [ ] **Step 2: Criar script de importação**

Criar `agents/sdlc-poc/tests/import-workflows.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

N8N_URL="http://localhost:5678"
WORKFLOWS_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/../workflows" && pwd)"

import_workflow() {
  local file="$1"
  local name
  name=$(python3 -c "import json,sys; print(json.load(open('$file'))['name'])")
  echo "Importing: $name"
  local resp
  resp=$(curl -s -X POST \
    -H "Content-Type: application/json" \
    -d "@$file" \
    "$N8N_URL/api/v1/workflows")
  local id
  id=$(echo "$resp" | python3 -c "import json,sys; d=json.load(sys.stdin); print(d.get('id', 'ERROR: ' + str(d.get('message','unknown'))))")
  echo "  -> Workflow ID: $id"
  echo "$id"
}

echo "=== Importing SDLC PoC workflows ==="
ID1=$(import_workflow "$WORKFLOWS_DIR/01-chat-discovery-to-spec.json")
ID2=$(import_workflow "$WORKFLOWS_DIR/02-spec-to-code.json")

echo ""
echo "=== Done ==="
echo "Workflow 1 (Chat) ID: $ID1"
echo "Workflow 2 (Code)  ID: $ID2"
echo ""
echo "Ative o Workflow 1 no n8n UI e acesse o chat em:"
echo "  http://localhost:5678/webhook/sdlc-poc-chat"
```

Criar `agents/sdlc-poc/tests/e2e-test.sh`:

```bash
#!/usr/bin/env bash
# End-to-end test for SDLC PoC: problem -> discovery -> spec -> code
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
N8N_WH="http://localhost:5678/webhook-test"
OUTDIR="$SCRIPT_DIR/generated-code"
mkdir -p "$OUTDIR"

echo "=== SDLC PoC End-to-End Test ==="
echo "Start: $(date)"
echo ""

PROBLEM="Build a REST API for a personal todo list manager. Users need to create, read, update and delete todo items. Each item has a title, optional description, and a done/not-done status. The API will be used by a mobile app."

echo "--- Phase 1: Discovery + Spec ---"
echo "Sending problem to workflow 1..."
START=$(date +%s)

PHASE1=$(curl -s -X POST "$N8N_WH/sdlc-poc/discovery" \
  -H "Content-Type: application/json" \
  -d "{\"problem\": $(python3 -c "import json; print(json.dumps('$PROBLEM'))")}" \
  --max-time 300)

END=$(date +%s)
echo "Phase 1 completed in $((END-START))s"

echo "$PHASE1" | python3 -c "
import json,sys
d=json.load(sys.stdin)
disc = d.get('discovery','')
spec = d.get('spec','')
print(f'  Discovery: {len(disc)} chars')
print(f'  Spec: {len(spec)} chars')
print(f'  Has hypotheses: {\"## Hypotheses\" in disc}')
print(f'  Has RF requirements: {\"RF-\" in spec}')
"

SPEC=$(echo "$PHASE1" | python3 -c "import json,sys; print(json.load(sys.stdin)['spec'])")

echo ""
echo "--- Phase 2: Spec to Code ---"
echo "Sending spec to workflow 2..."
START=$(date +%s)

PHASE2=$(curl -s -X POST "$N8N_WH/sdlc-poc/spec-to-code" \
  -H "Content-Type: application/json" \
  -d "{\"spec\": $(python3 -c "import json,sys; print(json.dumps(sys.stdin.read()))" <<< "$SPEC")}" \
  --max-time 300)

END=$(date +%s)
echo "Phase 2 completed in $((END-START))s"

echo "$PHASE2" | python3 -c "
import json,sys
d=json.load(sys.stdin)
files = d.get('files',[])
print(f'  Files generated: {len(files)}')
for f in files:
    print(f'    - {f[\"filename\"]} ({len(f[\"content\"])} chars)')
"

# Extract and save generated files
echo ""
echo "--- Extracting generated files to $OUTDIR ---"
echo "$PHASE2" | python3 -c "
import json,sys,os
d=json.load(sys.stdin)
outdir='$OUTDIR'
for f in d.get('files',[]):
    fname = os.path.basename(f['filename'])
    path = os.path.join(outdir, fname)
    with open(path,'w') as fh:
        fh.write(f['content'])
    print(f'  Saved: {path}')
"

# Try to run the tests if python3 + pip available
echo ""
echo "--- Attempting to run generated tests ---"
cd "$OUTDIR"
if command -v pip3 &>/dev/null; then
  pip3 install -q fastapi pydantic httpx pytest 2>/dev/null || true
  python3 -m pytest test_main.py -v --tb=short 2>&1 | tail -20 || echo "  Tests failed or not runnable — check output above"
else
  echo "  pip3 not available, skipping test run"
fi

echo ""
echo "=== End-to-End Test Complete: $(date) ==="
```

## Critérios de Sucesso do PoC

O PoC é considerado **viável** se ao menos 4 dos 5 critérios forem aprovados:

1. Discovery produz estrutura com hipóteses testáveis e métricas
2. Spec produz ≥5 functional requirements no formato RF-NN
3. Spec-to-Code gera ≥3 arquivos Python distintos
4. Ao menos 1 teste do código gerado passa sem modificação manual
5. Tempo total end-to-end < 5 minutos

Se PoC **aprovado** → próximo passo: adicionar Langfuse (Proposta F) para fechar o loop de observabilidade.

Se PoC **reprovado** → identificar a fase com menor qualidade, ajustar prompt correspondente em `agents/sdlc-poc/prompts/`, e re-executar apenas essa fase.
